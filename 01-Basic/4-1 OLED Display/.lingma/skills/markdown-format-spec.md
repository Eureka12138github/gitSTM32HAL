# Markdown 文档格式规范

## 🎯 技能定位

为 Markdown 文档生成提供详细的格式规范和操作指南，确保所有文档符合 markdownlint 标准。

## 适用范围

本规范适用于所有生成的 Markdown 文档：

- README.md - 项目概述
- 技术指南 - 各类使用教程
- CHANGELOG.md - 更新日志
- TROUBLESHOOTING.md - 故障排除
- QUICK_REFERENCE.md - 快速参考
- 其他 Markdown 格式内容

## 核心规则详解

### 1. 标题格式（MD022）

**规则：** 标题上下必须各有一个空行

✅ 正确：

```markdown
上一段内容

## 标题

下一段内容
```

❌ 错误：

```markdown
上一段内容
## 标题
下一段内容
```

### 2. 列表格式（MD032）

**规则：** 列表前后必须有空白行

✅ 正确：

```markdown
**标题：**

- 列表项 1
- 列表项 2

下一段文字
```

❌ 错误：

```markdown
**标题：**
- 列表项 1
- 列表项 2
下一段文字
```

### 3. 代码块格式（MD031, MD040）

**规则：**

- 代码块前后必须有空白行
- 必须指定语言标识

✅ 正确：

````markdown
上一段文字

```bash
代码内容
```

下一段文字
````

❌ 错误：

````markdown
上一段文字
```bash
代码内容
```
下一段文字
````

**语言标识规范：**

- Shell 命令：```bash
- 纯文本：```text
- INI 配置：```ini
- C 代码：```c
- PowerShell: ```powershell
- Mermaid 图表：```mermaid

### 4. 表格格式（MD060）

**规则：** 分隔线必须有空格，使用紧凑格式

✅ 正确：

```markdown
| 列名 1 | 列名 2 | 列名 3 |
| ------ | ------ | ------ |
| 内容   | 内容   | 内容   |
```

❌ 错误：

```markdown
| 列名 1 | 列名 2 | 列名 3 |
|--------|--------|--------|
| 内容   | 内容   | 内容   |
```

### 5. 标题末尾标点（MD026）

**规则：** 标题末尾不能使用标点符号（包括省略号）

✅ 正确：

```markdown
### 我想…
### 常见问题
### 快速开始
```

❌ 错误：

```markdown
### 我想...
### 常见问题：
### 快速开始！
```

### 6. 链接片段（MD051）

**规则：** 带 emoji 的标题在链接中必须包含 emoji

✅ 正确：

```markdown
## 🚀 快速开始

[快速开始](#-快速开始)
```

❌ 错误：

```markdown
## 🚀 快速开始

[快速开始](#快速开始)
```

## 完整规则清单

### 7. 强调作标题（MD036）

**规则：** 不能使用粗体代替正式标题

✅ 正确：

```markdown
##### 1. 首次配置（一次性）

或者

**答：** 直接回答
```

❌ 错误：

```markdown
**1. 首次配置（一次性）**
```

## ✅ 质量检查清单

在生成或修改任何 Markdown 内容前，必须逐项检查：

- [ ] **MD022** - 所有标题上下有空行
- [ ] **MD032** - 所有列表前后有空白行
- [ ] **MD031** - 所有代码块前后有空白行
- [ ] **MD040** - 所有代码块指定了语言标识
- [ ] **MD060** - 所有表格分隔线使用 `| --- |` 格式
- [ ] **MD026** - 所有标题末尾无标点符号
- [ ] **MD051** - 带 emoji 的标题链接包含 emoji
- [ ] **MD036** - 没有使用粗体代替正式标题

## 💡 最佳实践

### 表格格式技巧

✅ 推荐：

```markdown
| 参数 | 类型 | 说明 |
| ---- | ---- | ---- |
| `port` | int | 串口号 |
| `baud` | int | 波特率 |
```

❌ 避免：

```markdown
| 参数 | 类型 | 说明 |
|--------|--------|--------|
| `port` | int | 串口号 |
| `baud` | int | 波特率 |
```

### 代码块格式技巧

✅ 推荐：

````markdown
## 示例代码

下面是一个完整的 C 函数示例：

```c
void LED_Init(void) {
    // 初始化代码
}
```

接下来解释这个函数的作用...
````

❌ 避免：

````markdown
## 示例代码
下面是一个完整的 C 函数示例：
```c
void LED_Init(void) {
    // 初始化代码
}
```
接下来解释这个函数的作用...
````

### 列表格式技巧

✅ 推荐：

```markdown
**主要特点：**

- 支持自动同步
- 智能路径检测
- 备份机制

**注意事项：**

1. 首次使用需配置
2. 定期备份重要数据
3. 遇到问题查看 FAQ
```

❌ 避免：

```markdown
**主要特点：**
- 支持自动同步
- 智能路径检测
- 备份机制
**注意事项：**
1. 首次使用需配置
2. 定期备份重要数据
3. 遇到问题查看 FAQ
```
